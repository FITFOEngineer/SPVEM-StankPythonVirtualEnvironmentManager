⚠️ IMPORTANT: First-Time Setup on macOS

macOS Gatekeeper blocks unsigned scripts. You must do ONE of these before running:
  1. Right-click → Open on stank-venv-manager.command in Finder, then click "Open" in the dialog
  2. Or run this command in Terminal (recommended): 
    xattr -d com.apple.quarantine stank-venv-manager.command stank-venv-manager.sh; chmod +x stank-venv-manager.command stank-venv-manager.sh

This removes quarantine flag AND makes executable. Only needed once.

Why Python 3.11 (not 3.12+)?

▶

Python 3.11 is the recommended version for maximum package compatibility:

  • Mature ecosystem - All major data science/ML packages are thoroughly tested
  • No distutils issues - Python 3.12 removed distutils, breaking some packages
  • Widely supported - Libraries like Snowpark, some AWS tools require 3.11
  • Fast - 3.11 introduced 10-60% speed improvements over 3.10
  • Long support - Security updates until October 2027

Python 3.12 will work for most packages, but you may encounter occasional compatibility issues.

Why pip (not uv)

This tool uses pip instead of the newer uv package manager:

  • Pre-installed - pip comes with Python, no extra setup
  • Familiar - most tutorials and docs use pip
  • Stable - mature tooling, predictable behavior
  • Beginner-friendly - one less thing to install/learn

uv is 10-100x faster but requires separate installation. May add as optional mode in future versions.

💡 Tips & Tricks

Performance Optimization

Speed Up Package Installation

▶
  • Pip cache: Packages are cached after first download. Reinstalls are faster.
  • Wired connection: Use Ethernet instead of WiFi for large installs.
  • Off-peak hours: PyPI is faster when US/EU traffic is low.
# Clear pip cache if having issues:
pip cache purge

# Check cache size:
pip cache info

Maximize Apple Silicon Performance

▶
  • Use native ARM64 Python: Never run Python under Rosetta 2.
  • Enable MPS for PyTorch: Free GPU acceleration on M1/M2/M3.
  • Use Apple MLX: For LLM inference, MLX can be faster than PyTorch.
  • Unified memory advantage: No CPU-GPU transfer overhead.
# Verify native ARM64:
python3 -c "import platform; print(platform.machine())"
# Must print: arm64

# Check MPS:
python3 -c "import torch; print(torch.backends.mps.is_available())"

JupyterLab Power Tips

▶
  • Keyboard shortcuts: Press Cmd+Shift+C for command palette.
  • Multiple kernels: Each environment can be a separate kernel.
  • Dark mode: Settings → Theme → JupyterLab Dark.
  • Extensions: pip install jupyterlab-git for Git integration.
# Register environment as Jupyter kernel:
python -m ipykernel install --user --name=myenv --display-name="My Env"

# List available kernels:
jupyter kernelspec list

# Remove a kernel:
jupyter kernelspec remove myenv

Best Practices

Environment Management

▶
  • One project = one environment: Never share environments between projects.
  • Descriptive names: Use ml-customer-churn not env1.
  • Export regularly: Run pip freeze > requirements.txt after adding packages.
  • Don't modify system Python: Always use virtual environments.
  • Back up JupyterProjects: Your work is there, environments can be recreated.

Recovery - When Things Go Wrong

▶
  • Package conflict? Create a fresh environment instead of fixing.
  • Environment corrupted? Delete and recreate from requirements.txt.
  • Disk full? Clear pip cache: pip cache purge
  • Script hangs? Press Ctrl+C to cancel, then retry.
# Nuclear option - start fresh:
rm -rf ~/.venvs/broken-env
pip cache purge

# Recreate from requirements:
python3 -m venv ~/.venvs/fresh-env
source ~/.venvs/fresh-env/bin/activate
pip install -r requirements.txt

🔧 Troubleshooting

Installation Issues

"command not found: python" or "python3"

▶

Cause: Python not installed or not in PATH.

# Option 1: Install via Homebrew (recommended)
brew install [email protected]

# Option 2: Add alias to ~/.zshrc
echo "alias python='python3'" >> ~/.zshrc
source ~/.zshrc

# Option 3: Check installed Pythons
ls /opt/homebrew/bin/python*
which python3

"zsh: permission denied" when running script

▶

Cause: Script not marked as executable.

# Make both scripts executable:
chmod +x stank-venv-manager.sh
chmod +x stank-venv-manager.command

# Verify permissions (should show -rwxr-xr-x):
ls -la stank-venv-manager.*

"jq: command not found"

▶

Cause: jq (JSON processor) not installed. Required for this script.

# Install jq:
brew install jq

# If Homebrew not installed:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Add Homebrew to PATH (Apple Silicon):
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
source ~/.zprofile

# Then install jq:
brew install jq

Gatekeeper blocks script ("cannot be opened")

▶

Cause: macOS security blocking unverified scripts.

# Option 1: Right-click → Open in Finder (recommended)
# Click "Open" in the dialog

# Option 2: Remove quarantine attribute
xattr -d com.apple.quarantine stank-venv-manager.command
xattr -d com.apple.quarantine stank-venv-manager.sh

# Option 3: System Settings → Privacy & Security →
# scroll down → click "Open Anyway"

Package fails to compile (build errors)

▶

Cause: Missing Xcode Command Line Tools.

# Install Xcode CLI tools:
xcode-select --install

# If that fails, reset and reinstall:
sudo rm -rf /Library/Developer/CommandLineTools
xcode-select --install

# Accept license if needed:
sudo xcodebuild -license accept

# Install common build deps:
brew install openssl readline sqlite3 xz zlib pkg-config

Homebrew commands not found after install

▶

Cause: Homebrew not added to PATH.

# For Apple Silicon (M1/M2/M3):
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
source ~/.zprofile

# For Intel Mac:
echo 'eval "$(/usr/local/bin/brew shellenv)"' >> ~/.zprofile
source ~/.zprofile

# Verify:
which brew
brew --version

Runtime Issues

PyTorch not using GPU (MPS)

▶ rch.device("mps" if torch.backends.mps.is_available() else "cpu") model = model.to(device) tensor = tensor.to(device)

TensorFlow not using GPU (Metal)

▶

Cause: tensorflow-metal plugin not installed.

# Check GPU devices:
import tensorflow as tf
print(tf.config.list_physical_devices('GPU'))

# If empty, install Metal plugin:
pip install tensorflow-metal

# Verify:
python -c "import tensorflow as tf; print(tf.config.list_physical_devices())"

"mach-o file incompatible architecture"

▶

Cause: Mixing ARM64 and x86_64 binaries.

# Check Python architecture:
python3 -c "import platform; print(platform.machine())"
# Should print: arm64 (on Apple Silicon)

# If x86_64 on Apple Silicon, reinstall native:
brew uninstall [email protected]
arch -arm64 brew install [email protected]

# Clear cache and reinstall packages:
pip cache purge
pip install --force-reinstall numpy pandas

JupyterLab won't start or shows blank page

▶

Causes: Port conflict, browser cache, or extension issue.

# Try different port:
jupyter lab --port=8889

# Check what's using port 8888:
lsof -i :8888

# Kill stuck Jupyter processes:
pkill -f jupyter

# Clear Jupyter cache:
jupyter lab clean

# Reset settings:
rm -rf ~/.jupyter/lab/

JSON config not loading

▶

Cause: File missing or syntax error.

# Check file exists:
ls -la stank-venv-packages-macos.json

# Validate JSON syntax:
jq empty stank-venv-packages-macos.json

# Common JSON errors:
# - Trailing comma after last item
# - Missing quotes around strings
# - Single quotes instead of double
